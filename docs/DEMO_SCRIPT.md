# Demo Script

## 讲解结构

1. 介绍作品目标：粘贴 GitHub PR 链接即可自动审查，减少测试缺失、代码质量问题和高风险变更遗漏。
2. 展示主界面：左侧输入公开 GitHub PR URL，中间是 Review 报告，右侧是变更文件统计。
3. 粘贴一个公开 PR 链接并点击“分析 PR”，展示系统自动拉取标题、描述和 diff。
4. 展示审查意见：逐条讲解安全、测试、可靠性、可维护性和流程问题。
5. 展示 AI 代码评审：说明大模型会阅读 diff，评价代码质量、漏洞风险、测试充分性、考虑是否周到，并给出合并建议。
6. 展示报告导出：导出 Markdown Review 结果，说明可作为评审记录或 PR 评论草稿。
7. 展示浏览器插件：在 GitHub PR 页面点击“分析当前 PR”，页面内直接出现审查结果。
8. 介绍工程质量：核心逻辑有单元测试，项目支持 `npm run test`、`npm run build`、`npm run lint`。
9. 说明后续规划：支持 GitHub Token、私有仓库、Gitee、自动发布 Review 评论和团队规则。

## 录屏提示

- 录制时打开浏览器控制台不是必须，重点展示可操作流程。
- 视频中需要有声音讲解。
- 上传到 bilibili、云盘等外部可访问平台后，把链接写回 README 的 Demo 区域。
